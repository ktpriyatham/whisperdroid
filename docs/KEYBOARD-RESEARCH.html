Top 3 Recommendations

▼
#1 Best Overall

FlorisBoard

The perfect balance of modern tech and permissive licensing.

  • Why Speekez: Written in Kotlin, Apache-2.0 license, and extremely active. Its modular architecture makes adding a "Speekez Module" straightforward.
  • Specific Modules: app for UI, lib for IME logic.
  • Integration Effort: ~5-7 Days.
  • Risks: Currently in beta; some core features like word prediction are in flux.
#2 Best for Development

Thumb-Key

Modern Jetpack Compose architecture matches our current WhisperDroid stack.

  • Why Speekez: Uses Compose for everything. Very easy to build custom UI overlays for voice recording and transcription progress.
  • Specific Files: ThumbKeyIme.kt, KeyboardLayout.kt.
  • Integration Effort: ~3-5 Days.
  • Risks: AGPL-3.0 license is restrictive; unique grid layout might alienate "standard" users.
#3 Most Stable Base

HeliBoard

Best for users wanting a traditional Gboard-like experience.

  • Why Speekez: Solid AOSP base with modern features. Most feature-complete open-source keyboard today.
  • Specific Files: LatinIME.java, AudioManager.java.
  • Integration Effort: ~10-14 Days.
  • Risks: Heavy Java/C++ legacy; difficult to customize UI compared to Floris/Thumb. GPL-3.0 license.

Integration Strategy (#1 FlorisBoard)

▼

Action Plan

1
Fork & Modularize: Fork FlorisBoard. Create a :speekez-api module. Port our existing WhisperApiClient and ClaudeApiClient logic.
2
Smartbar UI: Locate SmartbarView.kt. Inject a microphone icon next to the clipboard icon. Add a longClick listener to trigger recording.
3
Transcription Hook: On audio release, send to Whisper. Use the EditorInstance to commit the returned text directly into the current field.
4
AI Refinement: If enabled, pass the text to Claude before committing. Show a "Refining..." status in the Smartbar.
